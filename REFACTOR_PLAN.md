# 流浪动物一站式领养管理系统重构方案（第一版）

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** 基于现有毕业设计项目重构出一个可在 IDEA 中直接打开运行的、审美与业务流程更现代的 Java + MyBatis + MySQL 项目《流浪动物一站式领养管理系统》。

**Architecture:** 采用 Spring Boot + MyBatis + MySQL 的经典三层架构，前台定位为“公众领养门户”，后台定位为“救助站运营工作台”。核心主线围绕“动物档案 → 医疗记录 → 可领养发布 → 领养申请 → 审核 → 协议/交接 → 回访闭环”。

**Tech Stack:** Java, Spring Boot, MyBatis, MySQL, Maven, IDEA。

---

## 一、对现有项目的判断

### 1. 现有项目的可复用部分
- Spring Boot + Maven 基础工程可参考
- MyBatis / MyBatis-Plus 的现有配置可作为迁移参考
- 文件上传、基础登录、通用分页响应等工具类可选择性复用
- 现有动物、领养、留言、论坛、公告等表可作为“旧数据来源参考”

### 2. 现有项目的主要问题
- 产品定位混乱，模块像模板拼接，业务主线不清晰
- 页面审美偏旧式课程设计风格，不符合你要求
- 数据模型过于 CRUD 化，缺少“状态流转”和“生命周期”设计
- 前后台存在写死地址、历史模板代码、无统一设计语言的问题
- 后台页面更多是“表单+表格”，缺乏真正的运营工作台体验

### 3. 重构原则
- 不继续在旧页面上修修补补，而是做“新项目骨架 + 旧数据/旧逻辑择优迁移”
- 后端保留 Java + MyBatis + MySQL，确保 IDEA 可直接开发维护
- 审美、信息架构、领域建模全部按新项目重做
- 优先实现 MVP 闭环，再逐步增强
